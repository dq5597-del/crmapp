# 光輝影音科技 CRM — 安裝部署說明

## 系統功能總覽

| 模組 | 功能 |
|------|------|
| 戰情室 | KPI 卡片、客戶狀態分布、近30天回訪提醒、最近報價單、花蓮7天天氣預報 |
| 客戶資料 | 新增/編輯客戶、聯絡人、拜訪紀錄、同業資訊、專案、報價單（各為子頁籤）|

| 銷貨單 | 從報價單轉入、可修改備註/狀態 |
| 訂購單 | 從報價單轉入、可填廠商資訊/修改狀態 |
| 廠商建檔 | CRUD 廠商資料、統編、銀行帳號、付款條件 |
| 庫存管理 | 庫存總覽、入庫/出庫/盤點異動紀錄 |
| 應收帳款 | 建立應收、登錄收款紀錄、逾期提醒、狀態追蹤 |
| 應付帳款 | 建立應付、登錄付款紀錄、顯示廠商匯款資訊、逾期提醒 |
| 產品管理 | 新增/編輯產品、定價/成本/利潤率、庫存、上下架 |
| 系統設定 | 公司資訊、匯款資料、報價單預設值、帳號角色管理 |

## 步驟一：建立 Supabase 專案

1. 前往 [https://app.supabase.com](https://app.supabase.com) → 建立新專案
2. 選擇區域：**Singapore**（離台灣最近）
3. 設定資料庫密碼並記下來
4. 進入 **SQL Editor** → 貼上 `supabase/schema.sql` 的全部內容 → 執行
5. 繼續在 **SQL Editor** → 貼上 `supabase/schema_additions.sql` 的全部內容 → 執行（廠商、庫存、應收帳款模組）
6. 繼續在 **SQL Editor** → 貼上 `supabase/schema_payables.sql` 的全部內容 → 執行（應付帳款模組）

### 建立 Storage Buckets

進入 **Storage** → 建立以下 Buckets（建議設為 Private）：
- `product-files`（型錄、說明書）
- `visit-photos`（拜訪現場照片）
- `quote-pdfs`（報價單 PDF）

## 步驟二：設定環境變數

1. 複製 `.env.local.example` → 改名為 `.env.local`
2. 填入 Supabase 資訊（從 Supabase → Project Settings → API）：

```env
NEXT_PUBLIC_SUPABASE_URL=https://你的專案ID.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...（anon public key）

# Resend 寄信（免費方案每月3000封）：https://resend.com
RESEND_API_KEY=re_你的key

# 老闆信箱（報價單寄到這裡）
BOSS_EMAIL=dq5597@gmail.com
```

## 步驟三：安裝與啟動

```bash
# 安裝依賴
npm install

# 本機開發
npm run dev
```

開啟 [http://localhost:3000](http://localhost:3000) — 第一次會導向到登入頁

---

## 步驟四：建立第一個帳號（管理員）

1. 進入 Supabase → **Authentication** → **Users** → **Invite User**
2. 輸入 Email（例：dq5597@gmail.com）→ 送出邀請
3. 收到邀請郵件 → 點擊連結設定密碼
4. 登入後，進到 **系統設定** → **帳號管理** → 將自己改為 **管理員**

後續新增其他帳號同樣透過 Supabase Dashboard Invite。

## 步驟五：部署到 Vercel（免費）

```bash
# 安裝 Vercel CLI（選用）
npm i -g vercel

# 部署
vercel --prod
```

或直接到 [https://vercel.com](https://vercel.com)：
1. Import Git repository
2. 在 **Environment Variables** 填入和 `.env.local` 相同的四個變數
3. Deploy

---

## 設定 Resend 寄信

1. 前往 [https://resend.com](https://resend.com) 註冊（免費）
2. API Keys → Create API Key → 複製填入 `RESEND_API_KEY`
3. 免費方案：每月 3,000 封，每天 100 封
4. **注意**：免費方案的 `from` 地址固定為 `onboarding@resend.dev`，如需自訂寄件地址需驗證自己的網域

## 常見問題

**Q：報價單寄信失敗？**  
A：確認 `RESEND_API_KEY` 正確，且 Resend 帳號已啟用。

**Q：天氣無法顯示？**  
A：Open-Meteo 是免費無需 API Key 的服務，若無法顯示可能是網路問題，重新整理即可。

**Q：Supabase 免費方案限制？**  
A：免費方案：500MB 資料庫、5GB Storage、50,000 MAU，一般小公司足夠使用。

**Q：如何增加使用者？**  
A：Supabase → Authentication → Users → Invite User。邀請後對方收 Email 設密碼即可登入。
